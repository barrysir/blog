Summary

This post is about writing a piecewise function as a single equation, using more conventional-algebraic functions than the usual Heaviside step function. Specifically it describes a way of condensing continuous piecewise functions into a single equation using only the absolute value function (which can be written out of conventional functions).

#2: Continuous, flat slices

Though the Heaviside method produces slices which become 0 outside of their interval, it's enough to have slices which are constant outside of their interval.

Let me notate s:a→bs: a \rightarrow bs:a→b as a slice function which is constant at y=ay=ay=a to the left and constant at y=by=by=b to the right. Say we wanted to add another slice, t:c→dt: c \rightarrow dt:c→d to the right of it.

  • For sss to not affect ttt's position, we have to subtract bbb from ttt. t−b:c−b→d−bt-b : c-b \rightarrow d-bt−b:c−b→d−b.
  • But for t−bt-bt−b to not affect sss's position, c−b=0c-b = 0c−b=0.

Therefore only slice functions that match up in this way s1:a→bs_1: a\rightarrow bs1​:a→b, s2:b→cs_2: b\rightarrow cs2​:b→c, s3:c→ds_3: c\rightarrow ds3​:c→d, ... can be connected. However, this doesn't say anything about the actual function piece, which can jump from aaa to wherever it needs to go, then connect/jump back to bbb when finished, so long as we're able to represent such a slice function within our constraints. I'll only consider continuous slices for now to keep things simple.

So we can join together constant slices by doing the following:

  • Sort them ascending by x-position.
  • Adjust each slice to start at y=0y=0y=0, besides the first one.

Surprisingly (dissapointingly) it's only one easy trick from the incline function to a slice of any function we want.

Bonus stuff

Introducing discontinuities

So far we've only been working with continuous piecewise functions, but is it also possible to do this with discontinuous piecewise functions?

The way we construct our slices from functions will allow for discontinuities within the function (for example, we can construct a slice of 1x\frac{1}{x}x1​ on [−1,1][-1,1][−1,1]). However I don't think it's possible to have jump discontinuities between the pieces, for the following reason:

Say we had a function s(x)s(x)s(x) with 1 or more of these discontinuities, and there was a way to write it under our constraints. We can remove all of the discontinuities by shifting the pieces around to get sˉ(x)\bar{s}(x)sˉ(x). sˉ(x)\bar{s}(x)sˉ(x) now has no discontinuities between its pieces, so we can use our algorithm to find an equation for it. But now consider s(x)−sˉ(x)s(x) - \bar{s}(x)s(x)−sˉ(x). This would be some sort of step function, and we would have just found a representation of it following our constraints. I don't know how to prove that such a representation doesn't exist, but I can't find any evidence saying that it does, leading me to believe that it's impossible.

Unless there is a function with a naturally occuring jump discontinuity, I don't believe it's possible to create piecewises containing jump discontinuities using this method.

Periodic piecewise functions

Is there a trick we can use to create periodic piecewise equations?

After searching around I found a couple of interesting functions:

  • sin⁡−1(sin⁡(x))\sin^{-1}(\sin(x))sin−1(sin(x)) is a linear function which oscillates between [−π2,π2][-\frac{\pi}{2}, \frac{\pi}{2}][−2π​,2π​] with period 2π2\pi2π. This is actually listed on the Wikipedia article for triangle waves. cos⁡−1(cos⁡(x))\cos^{-1}(\cos(x))cos−1(cos(x)) is similar, oscillating between [0,π][0, \pi][0,π] instead.
  • The wikipedia page for sawtooth waves also lists the function tan⁡−1(cot⁡(x))\tan^{-1}(\cot(x))tan−1(cot(x)). However this function is undefined at each multiple of π2\frac{\pi}{2}2π​, so it doesn't work well enough for our purposes.

Ideally we'd like a sawtooth shape for our periodic functions, but I can't find one and I have a hunch that there isn't one based on the problems encountered with discontinuities. As far as I know there are no other elementary periodic functions besides the trigonometric ones, so any search for more periodic functions would have to involve a trigonometric term. Combining two continuous functions (+, -, x, / by nonzero, composition) will always gives you a continuous function, so a discontinuous function would also have to be involved (sqrt, log, 1/x, tan, asin, acos).

For now, we can only make oscillating periodic functions. By exchanging xxx with sin⁡−1(sin⁡(x))\sin^{-1}(\sin(x))sin−1(sin(x)) and performing some other transformations, we can make an oscillating version of f(x)f(x)f(x), where [a,b][a,b][a,b] within f(x)f(x)f(x) gets mapped to [c,d][c,d][c,d] in the output (see the source code for details).
